Partial Dependency

Partial dependency occurs when non-prime attribute is functionally dependant on part of a candidate key

Lets Understand partial dependency shall we

In order to understand partial dependency, let us first know some basic terminologies with the help of an example.

Consider a relation(table) having four attributes P,Q,R,SP,Q,R,SP,Q,R,S having the following dependencies:

P,Q−>SQ−>RP,Q -> S\\ Q -> RP,Q−>SQ−>R

Using PPP and QQQ, we can derive SSS, and using QQQ, we can derive RRR. Hence, we can say that if we use both PPP and QQQ together we can derive all the attributes of the table. i.e, P,Q,R,SP,Q,R,SP,Q,R,S.(since P−>PP -> PP−>P and Q−>QQ -> QQ−>Q is self explanatory).

We can write PQ={P,Q,R,S}PQ = \{P,Q,R,S\}PQ={P,Q,R,S}, or in simple words, we can say the closures of PPP and QQQ give us all the attribute of the relation.. The minimal sets like PQPQPQ in a relation(table) that are capable of deriving all the attributes of a relation(table) are called Candidate keys. There can be more than one candidate key in a table.

If an attribute is a part of any candidate key of the relation, then it is called a Primary attribute else, it is said to be a Non-Primary attribute. In the example above, we can say that PPP and QQQ are primary attributes, and RRR and SSS are non-primary attributes.

We now know the basic definitions required to understand the concept of partial dependency. In the above example, SSS is dependent on all the primary attributes, i.e.,PPP and QQQ. If either PPP or QQQ are missing, then we cannot derive SSS. In the case of RRR, it is not the same.

Even if PPP, a primary attribute, is missing, we can still derive RRR using only SSS. Hence, instead of depending totally on the candidate key, RRR is partially dependent on QQQ, part of a candidate key. This is the concept of partial dependency.

What Causes Partial Dependency to Occur?

As we saw in the above section, partial dependency occurs whenever a non-prime attribute depends functionally on a part of the given candidate key.

In other words, Partial Dependency occurs when an attribute in a table depends on only a part of the primary key and not the whole key.

A functional dependency denoted as X→YX→YX→Y where XXX and YYY are an attribute set of a relation, is a partial dependency , if some attribute A∈XA∈XA∈X can be removed and the dependency still holds.
